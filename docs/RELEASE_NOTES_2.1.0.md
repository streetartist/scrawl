# Scrawl 2.1.0 版本说明

发布日期：2026-05-02

Scrawl 2.1.0 是在 2.0.5 正式发布之后的一次运行时补强版本。本次更新的重点不是继续扩展表层 API，而是把已经暴露出来、但此前没有真正落到原生运行时的关键能力继续接通，让 v2 在实际做游戏时更可用。

本版本的核心目标有三点：

1. 继续补齐 Python API 到 Rust/Bevy 运行时之间的真实链路。
2. 修复示例工程中的资源与路径问题，让 demo 更稳定可运行。
3. 将项目整体版本号统一提升到 2.1.0，保持发布链一致。

## 版本亮点

### 1. 运行时事件与文本能力继续补齐

本次版本进一步打通了此前常见但未完整落地的行为：

- 广播消息通过 Python 命令队列进入运行时。
- `say()` 和持久文本显示已接入原生渲染路径。
- 精灵删除时，关联文本实体会同步清理。
- 鼠标事件和精灵点击事件已接入运行时分发。

这意味着部分此前“API 可写但窗口里不生效”的行为，现在已经能真正跑起来。

### 2. 点击判定更加接近可用状态

为了让交互类游戏更稳定，本次对点击命中做了进一步增强：

- 支持旋转后的点击判定。
- 支持基于 mask 的点击命中判断。
- 精灵点击处理链条已接到 bridge 运行时。

这一改动对角色选中、按钮点击、射击类或指向类玩法都有直接帮助。

### 3. 音频系统完成基础接线

2.1.0 将音频相关 API 从“声明存在”推进到了“基础可运行”：

- Python `Game` 的音效和音乐接口已接入命令队列。
- bridge 运行时已能解析并执行音频命令。
- 音频插件已接入 bridge app。
- 背景音乐循环、暂停、恢复、停止的语义做了补正。

这让常见的背景音乐和音效触发场景不再停留在占位状态。

### 4. 渲染同步能力继续完善

本版本补齐了几条重要的运行时同步链：

- Scene 背景图现在会在启动阶段生成到场景中。
- Python 侧的画笔状态会同步到原生 `PenState`。
- 画笔落笔后，精灵移动会继续累积路径。
- Python 侧精灵颜色会同步到原生渲染颜色。

这使得背景图、画笔绘制和动态染色这些高频能力的行为更加符合预期。

### 5. Demo 目录与资源路径更稳了

本次对女巫 demo 做了实际可运行层面的整理：

- `demo_witch_v2.py` 已迁入 `demo_v2/`。
- 相关资源文件已整理到统一目录中。
- demo 的资源路径已改为脚本相对路径，不再依赖当前工作目录。

这意味着从不同 cwd 启动示例时，资源查找更不容易出错。

### 6. 兼容层方向已明确收缩

此前尝试过的 `compat.py` 兼容层方案，在本版本中已被移除，不再作为当前主线。

当前路线更明确：

- 优先补真实运行时能力。
- 不优先堆叠低价值包装层。

这能减少维护噪音，让版本推进更聚焦于“真的能做游戏”的部分。

### 7. 发布版本号统一提升到 2.1.0

本次还同步完成了版本号提升：

- Python 包元数据已切换到 2.1.0。
- Rust workspace 版本已同步切换到 2.1.0。
- 原生模块导出的 `__version__` 已切换到 2.1.0。
- 独立 app 的启动日志版本号已同步更新。
- `uv.lock` 中的项目版本已同步更新。

这保证了包元数据、原生模块、工作区和发布链对外口径一致。

## 文档更新

为避免后续继续重复审计，本版本新增了一份当前缺口说明：

- `docs/SCRAWL_V2_UNIMPLEMENTED.md`

该文件记录了截至 2.1.0 仍未真正接入运行时的主要模块和路径，包括：

- bridge 启动仍主要围绕 sprites
- 通用节点树运行时尚未完整落地
- physics / tilemap / ui / particles / navigation 仍需继续接线
- 原生 stub 仍有残留
- 独立 app 插件装配仍未完全放开

## 对用户的实际影响

如果你是引擎用户，这次更新意味着：

- 常见交互、文本、音频、背景和画笔行为比 2.0.5 更接近“可直接做游戏”的状态。
- v2 demo 的运行稳定性更好，资源路径问题更少。
- 部分之前写了 API 却没有运行结果的能力，现在已经开始真正生效。

如果你是维护者，这次更新意味着：

- 后续重点应继续放在 bridge 实体生成、节点树运行时与多模块真实接线上。
- `docs/SCRAWL_V2_UNIMPLEMENTED.md` 可以作为下一阶段补齐工作的基线清单。

## 已知事项

- 2.1.0 仍不是“所有导出模块都已完整运行”的终点版本。
- 当前 bridge 启动路径仍以 sprite 为核心，通用节点树还需继续推进。
- Physics、TileMap、UI、Particles、Navigation 等模块仍需要进一步接入原生运行时。
- `py_scene.rs` 的 `broadcast()` 与 `py_sprite.rs` 的 `say()` 在 native 层仍保留 stub。

## 总结

Scrawl 2.1.0 的价值，在于它继续把 v2 从“API 看起来很多”推进到“关键能力真的能跑”。

这不是一次大范围扩表的版本，而是一次围绕运行时打通、示例修正和发布统一性的工程化迭代。对于后续 2.x 版本，这是一轮必要而且正确的补课。
